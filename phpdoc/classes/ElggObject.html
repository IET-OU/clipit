    Elgg Object

    Elgg objects are the most common means of storing information in the database. They are a child class of ElggEntity, so receive all the benefits of the Entities, but also include a title and description field. An ElggObject represents a row from the objects_entity table, as well as the related row in the entities table as represented by the parent ElggEntity object.
    internal Title and description are stored in the objects_entity table.

    Load or create a new ElggObject.

    __construct(mixed $guid) 
    If no arguments are passed, create a new entity. If an argument is passed, attempt to load a full ElggObject entity. Arguments can be: - The GUID of an object entity. - A DB result object from the entities table with a guid property

    Parameters

    $guid

    mixed

    If an int, load that GUID. If a db row, then will attempt to load the rest of the data.

    Exceptions

    \IOException If passed an incorrect guid
    \InvalidParameterException If passed an Elgg* Entity that isn't an ElggObject
